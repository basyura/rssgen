# 複数サイトの RSS 統合計画

## 目的

設定ファイルの `feeds` に指定した複数サイトから更新情報を取得し、一つの
RSS 2.0 フィードとして出力する。

## 現状

- 各 `feeds` から更新情報を取得できる。
- 複数の `feeds` を指定すると、サイトごとに独立した RSS XML を順番に
  出力する。
- `-output` は `feeds` が一件の場合に限って利用できる。

## 修正案

- `settings.channel` に統合後の RSS チャンネル情報を定義する。
  - `title`: チャンネルのタイトル
  - `link`: チャンネルの URL
  - `description`: チャンネルの説明
- すべての `feeds` から取得した記事を一つの一覧にまとめる。
- 記事 URL をキーとして、サイトをまたいだ重複記事を除外する。
- 統合した記事一覧から一つの RSS XML を生成する。
- `-output` を `feeds` の件数にかかわらず利用可能にする。
- 各 `feeds` の `title` は取得元を識別できる情報として保持する。
  RSS item のタイトルには付加せず、既存の出力形式を維持する。
- 一部のサイト取得に失敗した場合の扱いは、現在の失敗時終了を維持する。
- `config.example.yml` を統合チャンネル形式の設定例として更新する。
- `README.md` の説明と実行例を、一つのフィードを生成する仕様に更新する。

## テスト方針

- 複数の設定から集めた記事が一つの RSS チャンネルに含まれることを確認する。
- 異なるサイトに同じ記事 URL がある場合に重複しないことを確認する。
- `settings.channel` の各項目が RSS チャンネルへ反映されることを確認する。
- 複数の `feeds` と `-output` を併用できることを確認する。
- 必須設定が不足している場合のエラーを確認する。

## 検証

- `gofmt -w main.go main_test.go`
- `go test ./...`
- `go build .`
- `go run . -config config.example.yml` を実行し、一つの RSS XML が
  出力されることを確認する。
- `go run . -config config.example.yml -output feed.xml` を実行し、
  統合された RSS XML がファイルへ出力されることを確認する。

## 実施結果

- すべての `feeds` から取得した記事を、一つの RSS チャンネルへ統合した。
- 記事 URL をキーとして、サイトをまたいだ重複を除外するようにした。
- `settings.channel` の `title`、`link`、`description` を RSS チャンネルへ
  反映するようにした。
- 複数の `feeds` を指定した場合も `-output` を利用可能にした。
- `config.example.yml` と `README.md` を統合フィードの設定・動作に合わせて
  更新した。
- 複数サイトの統合、重複除外、チャンネル設定、ファイル出力のテストを
  追加した。
- `go test ./...` と `go build -o /private/tmp/rssgen .` は成功した。
- `go run . -config config.example.yml -output
  /private/tmp/rssgen-feed.xml` は成功した。
- 実サイトから生成した XML は RSS と channel が各一件で、XML 構文検証も
  成功した。

## クラスセレクター対応修正案

- `.news-index` のようなクラス名だけの指定を XPath に変換する。
- `.news-index` は、タグを限定せず `news-index` クラスを持つ要素を
  抽出対象とする。
- 抽出対象の直接の子要素自身がリンクの場合も RSS item として扱う。
- `tag.class` と通常の XPath 指定は従来どおり利用可能とする。
- クラス名だけの指定に対する単体テストを追加する。
- `config.example.yml` を使い、L'Arc-en-Ciel の更新情報が抽出されることを
  実サイトで確認する。

## クラスセレクター対応実施結果

- `.news-index` をタグ未指定のクラスセレクターとして XPath に変換した。
- 抽出対象の直接の子要素が `<a>` の場合も記事として取得できるようにした。
- クラスセレクター変換と、直接の子リンク抽出の単体テストを追加した。
- `go test ./...` と `go build -o /private/tmp/rssgen .` は成功した。
- `config.example.yml` で実サイトを確認し、L'Arc-en-Ciel の記事を二十件
  抽出できることを確認した。

## サイト別記事数制限修正案

- 一つの URL から取得する記事を最大五件に制限する。
- HTML に記載されている順序を維持し、先頭から五件を採用する。
- 同じ URL の記事は一件として数え、重複を除いた記事が最大五件になるように
  する。
- 複数サイトを指定した場合、各サイトから最大五件ずつ取得して一つの RSS に
  統合する。
- 五件を超える記事と、五件未満の記事に対する単体テストを追加する。

## サイト別記事数制限実施結果

- 一つの URL から抽出する記事を、重複を除いた先頭五件に制限した。
- 六件以上の記事と重複 URL を含む HTML の単体テストを追加した。
- `go test ./...` と `go build -o /private/tmp/rssgen .` は成功した。
- `config.example.yml` で実サイトを確認し、二サイトから各五件、合計十件の
  RSS item が生成されることを確認した。

## RSS item タイトル整形修正案

- RSS item のタイトルを `取得元タイトル - 日付 : 記事タイトル` の形式に
  する。
- 取得元タイトルには `feeds[].title` を使用する。
- B'z の記事は `.date` と `.comment` から日付と記事タイトルを取得する。
- L'Arc-en-Ciel の記事は `<h5>` と `.NW-R` から日付と記事タイトルを
  取得する。
- 日付を取得できない場合は `取得元タイトル - 記事タイトル` とする。
- HTML 抽出と RSS タイトル出力の単体テストを追加する。

## RSS item タイトル整形実施結果

- B'z の `.date` と `.comment` から日付と記事タイトルを抽出した。
- L'Arc-en-Ciel の `<h5>` と `.NW-R` から日付と記事タイトルを抽出した。
- RSS item のタイトルを
  `取得元タイトル - 日付 : 記事タイトル` の形式にした。
- 日付がない場合は `取得元タイトル - 記事タイトル` とする処理を追加した。
- 日付・記事タイトルの抽出と、タイトル整形の単体テストを追加した。
- `go test ./...` と `go build -o /private/tmp/rssgen .` は成功した。
- 実サイトから生成した RSS で、両サイトの記事タイトルが指定形式になることを
  確認した。
